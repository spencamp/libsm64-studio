# LibSM64 Studio media capture guide

Use this guide for release screenshots and demonstration footage. Every asset
should present LibSM64 Studio as a Blender performance-capture tool, not as a
Fast64 terrain-testing add-on.

## Screenshots

1. **Ready to perform.** Show a composed Blender scene with Live Mario at a
   spatial Start Mark. Keep the **LibSM64 Studio** sidebar visible, including
   the **Mario Performance Studio** heading and **Record a Mario Performance**
   controls.
2. **Capture setup.** Show a saved Timeline Start Frame and a meaningful camera
   view. This explains that spatial and timeline marks are independent.
3. **Take review.** Show an active baked take in the timeline alongside the
   **Active Take**, **Favorite Takes**, and **Other Takes** sections. Use a
   frame where the performance reads clearly in the viewport or render.

## Demonstration footage

Record a focused 10–15 second GIF or video:

1. Open the **LibSM64 Studio** tab and start Live Mario in a finished Blender
   shot.
2. Rehearse briefly, set a Mario Start Mark and Timeline Start Frame, then
   begin recording.
3. Perform a short, readable movement through the shot and stop to bake it.
4. Scrub or play the baked take in Blender, ending on the rendered or material
   preview result.

Avoid using a terrain-editor viewport, collision diagnostics, or Fast64-only UI
as the hero image. Mention Fast64 only when a scene's compatibility context is
useful. Caption media with outcome-oriented language such as “Record a Mario
performance, bake the take, and render it in Blender.”
